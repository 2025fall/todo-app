# Todo应用云端部署计划 (国内版)

## 项目概述
基于Vue3 + FastAPI的全栈待办事项管理应用，支持PWA功能。

## 部署架构 (国内可访问)
- **前端**: Vue3 + TypeScript + Vite + PWA
- **后端**: FastAPI + SQLite + Uvicorn
- **部署平台**: 
  - 前端: Gitee Pages (国内访问稳定)
  - 后端: 腾讯云CloudBase (国内云服务)

## 平台选择原因

### Gitee Pages
- ✅ 国内访问速度快
- ✅ 免费静态网站托管
- ✅ 支持自定义域名
- ✅ 与GitHub同步方便
- ✅ 无需VPN或国外信用卡

### 腾讯云CloudBase
- ✅ 国内云服务，访问稳定
- ✅ 支持Python/FastAPI
- ✅ 免费额度充足
- ✅ 支持微信/QQ登录
- ✅ 完善的中文文档

## 部署步骤

### 1. 前端部署 (Gitee Pages)
**平台**: Gitee Pages
**配置**:
- 仓库: 从GitHub同步到Gitee
- 分支: `feature/pwa-mobile`
- 构建目录: `frontend/dist`
- 构建命令: `cd frontend && npm install && npm run build`

### 2. 后端部署 (腾讯云CloudBase)
**平台**: 腾讯云CloudBase
**配置**:
- 运行时: Python 3.9
- 入口文件: `backend/app/main.py`
- 安装命令: `pip install -r requirements.txt`
- 启动命令: `python -m uvicorn app.main:app --host 0.0.0.0 --port 9000`

## 环境变量配置

### 后端环境变量 (CloudBase)
```
DATABASE_URL=sqlite:///./todos.db
SECRET_KEY=your-secret-key-here
CORS_ORIGINS=https://your-username.gitee.io/todo-app
PORT=9000
HOST=0.0.0.0
RELOAD=false
CORS_ORIGIN_REGEX=https://.*\.gitee\.io
```

### 前端环境变量 (Gitee Pages)
```
VITE_API_BASE_URL=https://your-env-id.tcloudbaseapp.com/api
VITE_APP_TITLE=Todo应用
VITE_APP_DESCRIPTION=基于Vue3和FastAPI的待办事项管理应用
```

## 部署流程

### 阶段1: 准备工作
1. 注册腾讯云账号 (支持微信登录)
2. 注册Gitee账号
3. 从GitHub导入项目到Gitee

### 阶段2: 后端部署 (CloudBase)
1. 登录腾讯云控制台
2. 开通CloudBase服务
3. 创建新环境
4. 上传代码或连接Git仓库
5. 配置环境变量
6. 部署并获取API域名

### 阶段3: 前端部署 (Gitee Pages)
1. 在Gitee中开启Pages服务
2. 选择`feature/pwa-mobile`分支
3. 设置构建目录为`frontend/dist`
4. 配置自动构建
5. 部署并获取前端域名

### 阶段4: 跨域配置
1. 更新后端CORS_ORIGINS为Gitee Pages域名
2. 重新部署后端
3. 测试前后端连接

# 移动端PWA与云端部署计划

## 🎯 项目目标
- 提供可安装的PWA体验，支持离线使用和在线同步
- 将前端部署到全球CDN，自动提供HTTPS支持
- 将FastAPI后端部署到托管平台，支持持久化存储和环境管理
- 保持桌面端工作流程，同时让任何网络的用户都能访问应用

## 🏗️ 目标架构
- **前端** → Vercel静态托管（从`frontend`构建到`dist`）  
  提供CDN、HTTPS、预览部署，以及与GitHub的轻松集成
- **后端** → Railway服务（FastAPI + PostgreSQL）  
  Railway处理容器运行时、HTTPS端点、环境变量和托管PostgreSQL
- **数据库** → Railway PostgreSQL（替换本地SQLite）  
  通过`DATABASE_URL`环境变量暴露连接字符串
- **环境变量**
  - `VITE_API_BASE_URL`（前端）→ 指向Railway后端URL
  - `DATABASE_URL`、`SECRET_KEY`、`CORS_ORIGINS`、`PORT`（后端）→ 在Railway中设置
